# Внесение вклада в Scoria

[English](CONTRIBUTING.md)

Спасибо за интерес к Scoria! Этот документ описывает процесс внесения вклада и принятые соглашения.

## Настройка окружения разработки

```bash
git clone https://github.com/syn7xx/scoria.git
cd scoria

# Установите Rust, если ещё не установлен
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Установите системные зависимости
make deps

# Соберите проект
make build
```

## Стиль кода

- **Форматирование**: Запускайте `make fmt` перед коммитом. Используется `cargo fmt` с настройками по умолчанию.
- **Линтинг**: Запускайте `make check` для проверки предупреждений clippy. CI отклонит PR с предупреждениями.
- **Тестирование**: Вся новая функциональность должна быть покрыта тестами. Запускайте тесты через `cargo test`.

## Структура проекта

```
src/
├── main.rs          # Точка входа, разбор аргументов CLI
├── lib.rs           # Корень библиотеки, re-exports
├── app/tray/        # Реализация системного трея
├── engine/          # Основная логика: буфер, конфиг, горячие клавиши, хранилище, автозапуск, обновления
├── i18n/            # Интернационализация (английский + русский)
└── ui/              # Платформенный UI (GTK на Linux, AppKit на macOS, Win32 на Windows)
```

Ключевые модули в `engine/`:
- `clipboard.rs` — чтение буфера обмена (текст + изображения)
- `vault.rs` — запись в хранилище Obsidian
- `config.rs` — работа с файлом конфигурации
- `hotkey.rs` — регистрация глобальных горячих клавиш (X11; на Windows обычно используют PowerToys/AHK)
- `autostart.rs` — управление автозапуском
- `update.rs` — проверка обновлений + самообновление (Linux/macOS); на Windows обновление вручную через MSI/winget

## Отправка изменений

1. **Форкните** репозиторий и создайте ветку для функциональности от `main`
2. Внесите изменения, следуя соглашениям о стиле кода
3. Добавьте тесты для новой функциональности
4. Убедитесь, что `make check` и `cargo test` проходят
5. Зафиксируйте с понятным сообщением, описывающим **почему** (не только что)
6. Откройте Pull Request

CI запускается для pull request и push в `main` (`.github/workflows/ci.yml`) и проверяет форматирование, предупреждения clippy, тесты и кроссплатформенную smoke-сборку.

### Сообщения коммитов

Мы используем [Conventional Commits](https://www.conventionalcommits.org/). Формат:

```
<тип>(<область>): <описание>

[опциональное тело]
```

Типы: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.

Пример:
```
feat(clipboard): поддержка изображений WebP

- Добавлен WebP в список поддерживаемых форматов изображений
- Добавлен декодер с использованием crate `png` (WebP в PNG-контейнере)
```

## Сообщение об ошибках

При сообщении об ошибке укажите:
- Версию Scoria (`scoria --version`)
- ОС и окружение рабочего стола
- Шаги для воспроизведения
- Соответствующий вывод логов (запустите с `RUST_LOG=debug`)

## Запросы функций

Откройте issue с:
- Чётким описанием функции
- Вариантом использования (зачем это нужно?)
- Опционально: примерной идеей реализации
